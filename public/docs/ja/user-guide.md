# ArchivistPro ユーザーガイド

> バージョン 2.4.4 | 2026-05-05

## ようこそ

ArchivistPro は、建築関連ファイルを賢くアーカイブし、検索・管理するためのデスクトップアプリケーションです。

---

## 1. 初回起動 — セットアップウィザード

アプリケーションを初めて開いたとき、**セットアップウィザード (Setup Wizard)** が表示されます。4 つのステップでシステムを準備します:

1. **システムチェック** — お使いのコンピューターの互換性が検証され、表示言語が選択されます (トルコ語と英語は完全対応。中国語/日本語/アラビア語のインターフェースは部分的に翻訳されています)
2. **ハードウェア検出** — CPU とメモリが分析され、適切なパフォーマンスモード (Low / Medium / High) が推奨されます
3. **AI セットアップ** — AI 機能の動作方法を選択します:
   - **ローカル AI (Ollama)** — すべてのデータがコンピューター内に保持されます
   - **クラウド AI** — インターネット経由で動作します (API キーが必要)
   - **スキップ** — AI なしで開始し、後で設定 (Settings) から有効化できます
4. **サマリー** — 選択内容が表示され、セットアップが完了します

> **メモ:** ウィザードは一度だけ表示されます。初回起動時のウィザード後、管理者アカウント (admin) の作成を求められます (事前定義されたパスワードはありません — ご自身で設定します)。次回以降の起動では直接ログイン画面が表示されます。

## 2. メイン画面

メイン画面は 3 つのセクションで構成されています:

- **左パネル (サイドバー):** 上部にアーカイブセレクターとお気に入り、中央に検索ボックスとビジュアル検索の感度、下部に **ソースフォルダー (Source Folders)** とファセットフィルター (カテゴリー、プロジェクトフェーズ、素材など)
- **中央エリア:** ファイルリスト (Explorer、Dashboard、または Technical ビュー)
- **右パネル (詳細):** 選択したファイルの詳細

### 表示モード

| モード | 説明 |
|------|-------------|
| Explorer | カードベースのグリッドビューでファイルを表示します |
| Dashboard | 統計および分析チャート |
| Technical | テーブル形式の詳細リスト |

### ソースフォルダーパネル

左パネルのセマンティック検索セクションのすぐ下に **ソースフォルダー (Source Folders)** パネルがあります。ここでは:

- **スキャンしたすべてのルートフォルダーが自動的にリスト表示されます** (フォルダー名と、そのフォルダー内のファイル数とともに)
- フォルダーをクリックすると **フィルターとして適用されます** — そのフォルダー内のファイルだけが表示されます
- 複数のフォルダーを選択できます (OR ロジック: A または B または C)
- すべてのフィルターを解除するには「フィルターをクリア (Clear Filters)」を使用します
- フォルダー名の左側にある **▶ 矢印** をクリックすると、サブフォルダーツリーが展開され、ネストされたフォルダー構造が表示されます。任意のサブフォルダーをクリックしてフィルタリングできます
- 各フォルダーの隣にある **3 点メニュー** で次のアクションが可能です:
  - **名前変更 (Rename)** — フォルダーの表示ラベルを変更します
  - **再スキャン (Re-scan)** — そのフォルダーのみを再スキャンします (他のソースは変更されません)
  - **スキャンレポート (Scan Reports)** — 過去のスキャンでスキップまたはエラーになったファイルのリスト
  - **削除 (Remove)** — リストからフォルダーを削除します (ファイルはアーカイブに残ります)
  - **ファイルごと削除 (Delete with Files)** — フォルダーとそのすべてのアセットレコードを削除します

> **メモ:** ファイルが削除されたり新しいスキャンが実行されると、件数はリアルタイムで更新されます。BAK (バックアップ) ファイルはカウントから除外され、メイングリッドには表示されません。

## 3. ファイルスキャンとインデックス作成

1. 左パネルの **「フォルダーをスキャン & インデックス (Scan & Index Folder)」** ボタンをクリックします
2. スキャンしたいフォルダーを選択します
3. スキャンモードを設定します:
   - **リストに追加 (Add to List)(デフォルト):** 新しいファイルが既存のスキャンに追加されます
   - **ゼロから再スキャン (Re-scan from Scratch / Replace):** *選択されたフォルダー配下* の古いレコードのみが削除され、再スキャンされます。**他のソースフォルダーには触れません。**
   - **色抽出 (Color extraction):** 画像の主要な色を分析します (オプション)
4. **「スキャン開始 (Start Scan)」** をクリックします

スキャン中、モーダルのタイトルには **「ファイルをスキャン & インデックス (Scan & Index Files)」** と表示されます。

- 進捗バーと速度を追跡できます
- **一時停止/再開 (Pause/Resume)** はスキャンを一時的に停止します
- **キャンセル (Cancel)** はスキャンを完全に終了します

> **DWG ファイルについて:** ODA File Converter がインストールされている場合、レイヤー、ブロック、テキスト内容、xref が自動的に抽出され、詳細パネルに表示されます。ODA はバックグラウンドで非表示で実行されます。

> **単一のソースフォルダーを更新するには、** 「ソースフォルダー」パネルでそのフォルダーの 3 点メニューを開き、**「再スキャン (Re-scan)」** を選択します — モーダルが自動的に開き、そのフォルダーのみのスコープスキャンが実行されます。

### データセーフティ (チェックポイント)

スキャン中、データは定期的にディスクに保存されます。停電やクラッシュが発生しても、すでにスキャンされたファイルは失われません。チェックポイントの頻度は **設定 > ストレージ (Settings > Storage)** で変更できます (デフォルト: 50 ファイルごと)。

### フォルダー変更検出 (Watch)

設定で **「フォルダーの変更を監視 (Watch for folder changes)」** を有効にすると、スキャン済みのフォルダーでファイルが追加または変更されたときに通知が届きます。自動再スキャンを有効にすることもできます。

### マルチコアスキャンとハードウェア使用

ArchivistPro はスキャン処理を複数の CPU コアで並列実行します。初回セットアップ時に、アプリケーションはストレージタイプを自動検出し、適切なデフォルトワーカー数を選択します:

| ストレージタイプ | 推奨ワーカー数 |
|---|---|
| HDD (機械式) | 1 – 2 |
| SSD (SATA) | 3 – 4 |
| NVMe (≤ 8 論理コア) | 6 – 8 |
| NVMe (≥ 16 論理コア) | 10 – 16 |

ワーカー数は **設定 > ストレージ (Settings > Storage)** で変更できます。HDD ドライブでは、ワーカー数を増やしてもパフォーマンスは向上せず、逆にディスクヘッドの競合により速度が低下することがあります。

> **AI 機能 (セマンティック検索、RAG) について:** CPU の代わりに GPU が使用されます。これらの操作は別の Ollama サービスを介して実行され、ファイルスキャンを遅延させません。

## 4. 検索

### テキスト検索
検索ボックスに入力を開始してください。システムは 3 層の検索を実行します:
- **キーワードマッチ:** ファイル名、プロジェクト名、メタデータ、クライアント名、承認ステータス
- **セマンティック検索:** AI による意味ベースのマッチング (3 文字以上必要)
- **建築用語:** 用語が自動的に拡張されます (例: 「kitchen」→ 関連する概念)
- **あいまい検索 (Fuzzy search):** タイプミスを許容します — スペルミスでも正しい結果が返ります (4 文字以上の単語、最大 30% の誤差)

> **ヒント:** ハイフンを含むファイルコードを検索できます — 「A1-c3」や「A1-」のような短いコードが直接マッチします。

### ブール検索 (高度)

検索ボックスで論理演算子を使用できます:

| 演算子 | 例 | 意味 |
|----------|---------|---------|
| **AND** | `plan AND section` | 両方を含むファイル |
| **OR** | `kitchen OR bathroom` | 少なくとも 1 つを含むファイル |
| **NOT** | `project NOT old` | 「project」を含むが「old」を含まない |
| **"引用符"** | `"floor plan"` | このフレーズを正確に含む (単語が隣接し、順序通り) |

> 演算子は **大文字** で記述する必要があります (and ではなく AND)。組み合わせも可能です: `"floor plan" AND facade NOT draft`

### 高度な検索メニュー

トップバーの **「高度な検索 (Advanced Search)」** ボタンをクリックして、追加の検索タイプにアクセスします:
- **ビジュアル検索 (Visual Search)** — 画像をアップロードして類似のビジュアルを検索
- **シェイプ検索 (Shape Search)** — DWG/DXF ファイル内の幾何学的形状で検索
- **類似を検索 (Find Similar)** — 選択したファイルに類似するファイルをリスト表示
- **重複ファインダー (Duplicate Finder)** — 重複するファイルを検出

### ビジュアル検索
左パネルの画像アップロードボタンまたは高度な検索メニューから画像をアップロードします。類似するビジュアルが自動的に見つかります。

### 並べ替え

異なる基準で結果を並べ替えることができます:
- **マッチスコア (Match score)(デフォルト)** — 関連性の高い順から低い順
- **更新日 (Modified date)** — 新しい順または古い順
- **ファイル名 (File name)** — アルファベット順
- **ファイルサイズ (File size)** — 大きい順または小さい順

### フィルター
左パネルのファセットを使用して結果を絞り込みます:
- **カテゴリー:** 2D 図面、3D モデル、ドキュメント、レンダー、写真、テクスチャー、動画
- **プロジェクトフェーズ:** コンセプト、スケマティック、許認可、施工
- **承認ステータス:** ドラフト、レビュー中、承認済み、却下 (プロジェクトステータスセクションから割り当て)
- **素材グループ:** コンクリート、ガラス、金属、木材、石材、セラミック、コンポジット
- **カラーテーマ:** 暖色系、寒色系、モノクロ、アースカラー、パステル
- **建築様式:** モダン、ミニマリスト、インダストリアル、ブルータリスト、ネオクラシカル、オーガニック

### 日付範囲フィルター

左パネルのファセットの下に **日付範囲フィルター (date range filter)** があります。開始日と終了日を選択して、更新日でファイルを絞り込みます。たとえば、過去 3 か月に変更されたファイルだけを表示するために使用します。

### フィルタープリセット

頻繁に使用するフィルター組み合わせを **プリセット (presets)** として保存できます:
1. フィルターを希望どおりに設定します (ファセット + タグ + 日付 + 検索語)
2. フィルターバーの **「保存 (Save)」** ボタンをクリックします
3. プリセットに名前を付けます
4. 後でワンクリックで同じフィルターを再読み込みできます

> 複数のオプションをチェックできます (OR ロジック)。フィルターは即座に適用されます — 再スキャンは不要です。

## 5. ファイル詳細

ファイルをクリックすると、右パネルに詳細が表示されます:
- プレビュー (サムネイルまたはネイティブビュー)
- カラーパレット (Hex、RGB、HSL、RAL コード)
- メタデータ (サイズ、日付、レイヤー、ブロック)
- AI タグ
- ユーザータグ
- プロジェクトステータスフィールド
- リンクされたファイル

### 5.1 プロジェクトステータス

詳細パネルには、ファイルのプロジェクト情報を入力できるセクションがあります:

| フィールド | 説明 | 制限 |
|-------|-------------|-------|
| **クライアント (Client)** | ファイルに関連付けられたクライアントまたは会社名 | 最大 150 文字 |
| **承認ステータス (Approval Status)** | 4 つのステータスボタンから 1 つを選択 | ドラフト / レビュー中 / 承認済み / 却下 |
| **バージョン (Version)** | バージョンラベル (例: v1.0、Rev-A) | 最大 20 文字 |
| **納期 (Delivery Date)** | カレンダーから選択するか手動入力 | ISO 日付形式 |

- 任意のフィールドをクリックして編集モードに入ります。Enter または外側をクリックして保存します
- 文字数制限に近づくとカウンターがオレンジ色になります (クライアント: 140+、バージョン: 18+)
- このデータは **ファイルが再スキャンされても保持されます** — スキャナーはこれらのフィールドに触れません
- クライアント名と承認ステータスは **検索結果に影響** し、サイドバーでフィルタリングできます

#### 却下理由
ファイルの承認ステータスを **「却下 (Rejected)」** に変更すると、下部にテキストフィールドが表示されます。却下の理由を入力できます (例: 「寸法が間違っているため、修正が必要」)。後でファイルが承認されると、却下理由は自動的にクリアされます。

#### 承認履歴
すべての承認ステータスの変更が記録されます。管理者は Dashboard ビューの **「承認履歴 (Approval History)」** パネルから、最近のアクションを時系列で確認できます (誰が、いつ、どのステータスからどのステータスへ、理由がある場合は理由とともに)。

### 5.2 リンクされたファイル (ファイルリレーション)

詳細パネルの「リンクされたファイル」セクションには、ファイルと他のファイルとの関係が表示されます:

| タイプ | 意味 | 例 |
|------|---------|---------|
| **PDF 出力** | このファイルの PDF バージョン | plan.dwg ↔ plan.pdf |
| **レンダー** | このデザインのビジュアライゼーション | salon.max ↔ salon_render.jpg |
| **バージョン** | 同じファイルの異なるバージョン | plan_v1.dwg ↔ plan_v2.dwg |
| **プロジェクトグループ** | 同じプロジェクトに属するファイル | Tower_A/*.* |

- **自動検出:** 同じフォルダー内で同じ名前で異なる拡張子のファイル (例: plan.dwg + plan.pdf) は、スキャン後に自動的にリンクされます
- **自動バージョンクラスタリング:** スキャン中、似た名前のファイル (例: `plan_v1.dwg`、`plan_v2.dwg`、`plan_Rev-A.dwg`、`plan_FINAL.dwg`) が「バージョン」リレーションで自動的にリンクされます。認識されるパターン: `_v1`、`_Rev-A`、`_R01`、`_FINAL`、`_DRAFT`、`_old`、`_new`、`(Copy)`、末尾の数字
- **手動リンク:** 「リンク追加 (Add Link)」ボタンを使用して、任意の 2 つのファイル間にリレーションを作成します
- **ナビゲーション:** リンクされたファイル名をクリックすると、その詳細に移動します
- 自動検出されたリンクには `[Auto]` ラベルがマークされます

### 5.3 フォーマット固有の情報

#### 3ds MAX (.max) ファイル
- **レイヤー (Layers)** — ファイルのレイヤー構造が色付きラベルとして表示されます
- **オブジェクト (Objects)** — ファイル内のオブジェクト名がリストされます (30 を超える場合は「+N more」のメモが追加されます)
- **FBX / OBJ エクスポート** — 詳細パネルに 2 つの変換ボタンが表示されます:
  - **クイックモード (Quick Mode):** 3ds Max は不要、基本的なジオメトリ変換
  - **ネイティブモード (Native Mode):** お使いのコンピューターに 3ds Max がインストールされている場合のフルクオリティ変換 (自動検出)

#### Revit (.rvt) ファイル
- 3ds Max なしでサムネイルプレビューが自動抽出されます (ファイルに埋め込まれた OLE ストリームから)
- プロジェクト名、Revit バージョン、フロア名が表示されます

#### DWG ファイル (ODA を使用)
- レイヤー、ブロック、テキスト内容、xref
- 図面タイプ (平面図、断面図、立面図など — AI 利用)
- スケールと単位の推定
- **シェイプデータ** — ファイル内の幾何学的形状 (長方形、円、ポリラインなど) が抽出され、シェイプ検索で使用されます
- **構造類似性** — DWG を右クリックして「類似を検索」を選択すると、レイヤー/ブロック/テキスト/シェイプ構造を比較して、最も類似する DWG ファイルがランク付けされます (CLIP ビジュアル比較ではなく構造複合スコアを使用 — CAD ファイルにはより信頼できます)
- **埋め込み OLE オブジェクト** — DWG 内に埋め込まれた Excel、Word、または PDF ファイルが検出され、詳細パネルに表示されます

## 6. タグ

ファイルに独自のタグを追加できます:
1. ファイル詳細パネルのタグセクションに移動します
2. 新しいタグを作成するか、既存のものから選択します
3. タグに色を割り当てることができます
4. 一括タグ付け: 複数のファイルを選択して、すべてに一度にタグを割り当てます

## 7. お気に入りとコレクション

- **お気に入り (Favorites):** 頻繁に使用するファイルにマークを付けてクイックアクセス (詳細パネルの星アイコン)
- **コレクション (Collections):** ファイルをテーマグループに整理 (例: 「ファサードプロジェクト」、「レンダーアーカイブ」)
  - コレクションに色を割り当てることができます
  - 1 つのファイルが複数のコレクションに属することができます

## 9. AI 機能

### AI ステータスインジケーター

トップバーの右側に **脳のアイコン** があり、その横に色付きのドットがあります:

| ドットの色 | 意味 |
|-----------|---------|
| 緑 | AI 準備完了 — モデルがロードされ、稼働中 |
| 黄 | モデル不足 — AI セットアップが未完了 |
| 赤 | Ollama がオフ、または接続なし |

脳のアイコンにマウスを合わせると **「AI セットアップウィザードを開く (Open AI Setup Wizard)」** リンクが表示されます。アイコンをクリックすると AI 設定モーダルが開きます。

---

### AI セットアップウィザード

AI 機能を初めてセットアップするか、トラブルシューティングするには:

1. トップバーの **脳のアイコン** をクリック → AI 設定 → **「AI セットアップウィザード (AI Setup Wizard)」** ボタン
   *または* 脳のアイコンにホバー → **「AI セットアップウィザードを開く」** リンクをクリック

2. ウィザードは **3 つのステップ** でセットアップを完了します:
   - **1. Ollama チェック** — Ollama がインストールされ、稼働しているかチェックします。インストールされていない場合は、「開始 (Start)」ボタンでアプリから直接起動するか、[ollama.com](https://ollama.com) からダウンロードできます。
   - **2. モデルダウンロード** — 必要な AI モデル (テキストおよび/またはビジュアル) がダウンロードされます。ダウンロードの進捗はプログレスバーで追跡できます。
   - **3. 完了** — システムが準備完了です。AI チャットと自動分類機能が有効になります。

---

### Ollama の開始/停止

AI 設定パネルから Ollama サービスを手動で管理できます:

- **Ollama がオフの場合** → 緑色の **「開始 (Start)」** ボタンが表示されます (電源アイコン)。クリックすると、バックグラウンドで `ollama serve` が起動します。
- **Ollama がオンの場合** → 赤い **「停止 (Stop)」** ボタンが表示されます。クリックするとサービスが停止します。
- ステータス変更中は、ボタンに「開始中... (Starting...)」/「停止中... (Stopping...)」と表示されます。

> **メモ:** Ollama がオフのとき、高度なモデル設定セクションは非表示になります — これは通常の動作です。

### AI チャット (RAG)

トップバーの **「💬 AI」** ボタンをクリックして AI チャットパネルを開きます。アーカイブについて自然言語で質問できます。

**主な機能:**

- **引用付きの回答 (Citation):** すべての AI 応答には、どのファイルが使用されたかが表示されます。ソースファイル名をクリックすると、そのファイルの詳細に直接移動します。
- **スコープフィルター:** チャットウィンドウの上部から検索範囲を絞り込みます — 特定のプロジェクト、タグ、またはアーカイブ全体から選択します。
- **ストリーミング応答:** AI の回答は生成されながら画面にライブストリーミングされます。長い応答を待つ必要はありません。

**質問例:**
- 「木製ファサードのレンダーはありますか?」
- 「2024 年に承認されたすべての平面図をリストしてください」
- 「クライアント名が 'Smith' のプロジェクトの PDF を検索してください」

---

### スラッシュコマンド

チャットボックスで `/` で始まる特殊コマンドを入力すると、異なる検索モードがアクティブになります:

| コマンド | ショートカット | 機能 |
|---------|----------|----------|
| `/visual <text>` | `/v <text>` | CLIP モデルによるテキスト → ビジュアルセマンティック検索。例: `/v modern wooden facade` |

> **ヒント:** `/visual` コマンドは画像のコンテンツを解釈します — 標準のテキスト検索とは異なり、ファイル名ではなくビジュアルの類似性に基づいて結果を返します。

---

### 合成モード (Synthesis Mode)

合成モードを使用して、複数のドキュメントにまたがる比較分析を実行します:

1. チャットウィンドウの **📎 (クリップ) ボタン** をクリックします
2. 分析したいファイルを選択します (複数選択可能)
3. 質問を入力します — 例: 「これら 2 つの平面図を比較して、違いをリストしてください」

AI は選択したドキュメントを一緒に評価し、合成された応答を生成します。

### チャットエクスポート

チャット履歴を保存または共有するには:

- チャットウィンドウのタイトルバーの **ダウンロード (⬇) ボタン** をクリックします
- チャット全体が **Markdown (.md) 形式** でコンピューターに保存されます
- ソースファイルの参照と応答がフォーマットされてエクスポートされます

### AI タグ提案

ファイルが詳細パネルで開いているときに、**✨ (キラキラ) ボタン** をクリックします。AI がファイルの内容を分析し、適切なタグを提案します。任意の提案をワンクリックで追加できます。

### 自動分類

スキャン中、AI モデルは自動的に:
- ファイルを分類 (レンダー / 写真)
- 素材を検出
- 建築様式を識別

### Vision AI (オプション)
設定 > AI 設定からビジョンプロバイダーを選択して有効化:
- 図面タイプ検出 (平面図、断面図、立面図)
- 素材と要素分析
- OCR (テキスト認識)

### AI 感度フィルター (管理者)

アーカイブ内の一部のファイルには機密情報が含まれている場合があります (給与表、契約書、個人データ)。これらのファイルを AI チャットの結果として表示したくない場合は、**AI 感度フィルター (AI Sensitivity Filter)** を使用します。

**なぜ必要か?** AI チャットはスキャン済みのすべてのファイルを検索します。ユーザーが「給与表はありますか?」と尋ねたとき、アーカイブにそのようなファイルがあれば、AI はそれを見つけて表示します。フィルターはそのようなファイルを AI から非表示にします。

**3 つの保護方法:**

1. **既製のカテゴリー** — 設定 > セキュリティ > AI 感度フィルターから有効化:
   - **財務:** 給与、請求書、見積書、予算、進捗払い...
   - **個人情報:** 国民 ID、電話、住所...
   - **法務:** 契約書、NDA、委任状...
   - **人事:** 人事ファイル、休暇、記録、業績...

2. **カスタムキーワード** — 独自のキーワードを追加します (例: 機密のクライアント名、プロジェクトコード)

3. **ファイル/フォルダー単位** — ファイルを右クリック → 「AI から非表示 (Hide from AI)」、またはソースフォルダーメニューから → 「AI から除外 (Exclude from AI)」

> **メモ:** 非表示にされたファイルはアーカイブ内では引き続き表示されます — 検索、フィルタリング、詳細パネルは通常通り機能します。AI チャットだけがこれらのファイルにアクセスできなくなります。

## 9.3 XMP メタデータエクスポート

標準の XMP サイドカー形式でファイルのメタデータをエクスポートできます:

1. ファイルを右クリック → **「XMP エクスポート (Export XMP)」** を選択
2. ファイルの隣に `.xmp` 拡張子のサイドカーファイルが作成されます (例: `plan.dwg` → `plan.xmp`)
3. オリジナルの隣に書き込めない場合 (権限のない場所など)、自動的にアプリケーションデータフォルダーに保存されます

XMP ファイルには以下が含まれます: ファイル名、プロジェクト名、カテゴリー、タグ、承認ステータス、クライアント名、バージョンラベル、却下理由 (ある場合)。

> **何のためか?** Adobe Bridge や Lightroom などのツールは XMP ファイルを読み取れます。別の DAM ソフトウェアに移行する場合、メタデータは持ち運び可能です。

## 9.4 ヘルスチェック (フィクシティチェック)

アーカイブ内のファイルの整合性を検証するためのヘルススキャンを実行できます:

1. 設定の **「ヘルスチェック (Health Check)」** セクションに移動します
2. **「スキャン開始 (Start Scan)」** をクリックします
3. システムはファイルのサンプルベースのチェックサム検証を実行します
4. 変更または破損したファイルが報告されます

この機能は、特に大規模なアーカイブで **bit-rot (サイレントなデータ破損)** を検出するのに役立ちます。

## 9.5 マルチアーカイブ

ArchivistPro は複数のアーカイブを並列で管理できます。左パネル上部の「アーカイブ (Archive)」セクションからアーカイブを切り替えます。

### 固定アーカイブ
- **メインアーカイブ (shared)** — 管理者が管理する共有アーカイブ
- **ローカルアーカイブ (personal)** — あなただけがアクセスできる個人アーカイブ

### カスタムアーカイブ
管理者が作成した追加のアーカイブ (例: 「オフィスセントラル」、「タワープロジェクト」) は、同じタブ列に表示されます。各アーカイブには以下があります:

- ソースフォルダーリスト
- アセットコレクション
- タグとお気に入りプール
- スキャン設定

これらが個別に保持されます。アーカイブ間を切り替えると、左パネルのソースフォルダーとカウントが新しいアーカイブを反映するように自動的に更新されます。

### マージと抽出 (管理者)
管理者は 2 つのアーカイブをマージ (Join/Merge) するか、アーカイブからフィルター済みのサブセットを抽出 (Extract) できます。これらの操作の前に自動スナップショットが取得され、必要に応じて元に戻すことができます。

## 10. LAN 経由でアーカイブをダウンロード

管理者がローカルネットワーク経由で共有しているアーカイブをダウンロードできます。

### 接続
1. 設定 > ネットワークタブに移動します
2. 管理者から提供された情報を入力します:
   - **IP アドレス** (例: `192.168.1.106`)
   - **接続コード** (8 桁のコード)
3. 「接続 (Connect)」をクリックします

### ダウンロード
接続が成功した場合:
- サーバーのバージョンとデータベースサイズが表示されます
- 「アーカイブをダウンロード (Download Archive)」をクリックします
- ダウンロードが完了すると、アーカイブが自動的にロードされます
- ページ更新は不要です

### アーカイブインポート (ファイルから)
LAN 接続なしでアーカイブを受け取ることもできます:
1. 設定 > ネットワーク > **「インポート (.archivistpro)」** をクリックします
2. 管理者から提供された `.archivistpro` ファイルを選択します
3. マニフェストプレビューを確認します
4. 確認 — アーカイブがロードされます

> **メモ:** LAN ダウンロードとファイルインポートの両方が、現在のアーカイブを更新します。セッションは保持されます — 再ログインの必要はありません。

## 11. セキュリティとプライバシー

- すべての AI 操作は **ローカルコンピューター上** で実行されます (デフォルト)
- ファイルはどのサーバーにも送信されません
- クラウド AI (Gemini、OpenAI) は **有効化した場合のみ** 使用されます
- API キーはセッションベースで保持され、ディスクには保存されません
- 破壊的な操作 (削除、一括移動) には確認ダイアログが必要です

### パスワードを忘れた場合

パスワードを忘れた場合は、ログイン画面の **「パスワードを忘れた場合 (Forgot Password)」** リンクを使用します。管理者から `recovery.key` ファイル (またはその内容) を取得する必要があります。このキーで新しいパスワードを設定できます。

## 12. 重複 & 類似ファイルファインダー

このツールは、アーカイブ内の重複または類似のファイルを表示します。

> **viewer として:** スキャンを実行し、結果を表示できます。ファイル **削除は管理者に制限されています** — 削除ボタンは表示されません。

### パネルへのアクセス

トップバーの **⎇** アイコンをクリックします (ゴミ箱と脳アイコンの間)。ホバーすると「重複を検索 (Find Duplicates)」が表示されます。

### スコープ選択

パネルが開いたら、タブからスキャンするアーカイブを選択します (メインアーカイブ、ローカルアーカイブ、および追加したすべてのカスタムアーカイブがリストされます)。

> ローカルアーカイブがまだロードされていない場合、タブは非アクティブに見えます。サイドバーからローカルアーカイブに切り替えてから戻ります。

> **大規模なアーカイブ** (2000 以上のファイル) では、スキャン開始前に警告が表示されます — 選択したモードに応じてスキャン時間が異なります。スキャン中に **キャンセル** ボタンでスキャンを即座に停止できます。

### 検出モード

| モード | 検出するもの |
|------|--------------|
| **完全一致重複 (Exact Duplicate)** | 内容が同一のファイル (SHA-256 ハッシュマッチ — 名前/フォルダーが異なる場合あり) |
| **同名 (Same Name)** | 異なるフォルダー内の同じ名前のファイル (一般基準で絞り込み可能 — 例: 同じ名前 + 同じサイズ) |
| **ビジュアル類似 (Visual Similarity)** | 視覚的に類似する写真/レンダー/画像ファイル (pHash) |
| **構造類似 (Structural Similarity)** | 似たレイヤー、素材、コンテンツ構造を持つ CAD/3D/ドキュメントファイル (Jaccard + 複合スコア) |

### 類似性しきい値

スライダーを使用して、**ビジュアル** および **構造** の類似性の感度を設定します:

- **低 %** → より多くの、緩くマッチする結果 (誤検知の可能性あり)
- **高 %** → 少ない、非常に類似した結果のみ

### 高度な基準パネル

「高度な基準 (Advanced Criteria)」をクリックすると、**4 つのセクション** を持つパネルが開きます:

#### 1. 一般基準 (クロスフォーマット事前フィルター)
同名と構造類似モードに追加条件を加えます:

- **同じファイルサイズ** — 許容差: 完全一致、±1 KB、または ±1%
- **更新日が近い** — N 日のウィンドウ (1–365)
- **同じフォルダー名** — 親フォルダー名のマッチ (大文字小文字を区別しない)

> 例: 「同じ名前 + 同じサイズ」は非常に強い重複シグナルです — 誤検知を大幅に削減します。

#### 2. フォーマット固有の基準
構造類似性で比較されるメタデータフィールドを選択します。フォーマットでグループ化されています:

- **DWG / DXF:** レイヤー · ブロック · テキスト内容 · Xref
- **IFC:** フロア数 · エンティティ数
- **3DS MAX:** 素材リスト · レンダーエンジン · Max バージョン
- **SketchUp:** コンポーネント · レイヤー · SketchUp バージョン
- **Revit:** フロア名 · プロジェクト名
- **PDF / Office:** タイトル · 著者 · ページ数

#### 3. パフォーマンスフィルター
スキャンプールを絞り込んで速度を上げます:

- **最小ファイルサイズ (KB)** — このサイズ未満のファイルはスキャンから完全に除外されます

#### 4. フォーマット可視性フィルター
スキャン結果に表示するファイルカテゴリーをリアルタイムで調整します (再スキャン不要):

CAD · BIM/3D · ドキュメント · ビジュアル · 動画 · バックアップ

> 一般基準またはパフォーマンス設定が変更されると、パネル下部に「再スキャンが必要 (Re-scan required)」の警告が表示されます。

### 結果の表示

**リストビュー:**

```
▼ VISUAL SIMILAR — 1 group
  ┌─────────────────────────────────────────────────┐
  │ render_v1.jpg   pHash diff: 5/64 bits → 92%    │
  │  /Render/render_v1.jpg   2.4 MB   2024-03-01   │
  │  /Render/render_v2.jpg   2.4 MB   2024-03-15   │
  └─────────────────────────────────────────────────┘
```

グループをクリックして展開/折りたたみます。各ファイルのパス、サイズ、日付が表示されます。

### 比較ビュー

2 つのファイルを並べて検査するには、`[⎇]` アイコンをクリックし、上部の **[比較 (Compare)]** ボタンに切り替えます:

```
┌───────────────────┬───────────────────┐
│   render_v1.jpg   │   render_v2.jpg   │
│  [ preview ]      │  [ preview ]      │
│  2.4 MB           │  2.4 MB           │
│  2024-03-01       │  2024-03-15       │
├───────────────────┴───────────────────┤
│  Similarity: 92% · pHash Δ: 5/64 bit │
└───────────────────────────────────────┘
```

ファイルを削除したい場合は、管理者に支援を依頼してください。

## 13. アプリケーション終了確認
