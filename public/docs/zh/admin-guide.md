# ArchivistPro 管理员指南

> 版本 3.0.0 | 2026-05-23 — 本指南仅适用于管理员（具备完整权限）用户。

> **v3.0.0 管理员快速摘要:** 本版本引入了 **V3 架构** — 向量数据 (embeddings、文本片段、关联、FTS) 移至单独的 `archivist_vec.db` 文件。现有档案在首次启动时自动迁移（包含 `.bak` 备份）。请更新备份工作流以适应新的文件结构。详情见下方"4a. V3 架构"。

---

## 1. 用户角色与权限层级

ArchivistPro 设有三级权限层级：

| 角色 | 谁 | 简介 |
|------|----|----|
| 创始管理员 (Founder Admin) | 首次安装程序的人，第一个创建的管理员账户 | 可以做任何事 + 管理其他管理员 |
| 管理员 (Admin) | 由创始者添加的管理员 | 管理档案和 viewer 用户 |
| 普通用户 (Viewer) | 普通用户 | 档案只读，拥有自己的本地档案 |

### 权限表

| 操作 | 创始者 | 管理员 | 用户 |
|------|:------:|:------:|:----:|
| 查看档案 | YES | YES | YES |
| 写入 / 扫描档案 | YES | YES | NO |
| 添加 / 删除 viewer | YES | YES | NO |
| 添加新管理员 (admin) | YES | NO | NO |
| 删除 / 降级管理员 | YES | NO | NO |
| 管理应用程序设置 | YES | YES | NO |
| 查看日志 | YES | YES | NO |
| 创始者的角色 / 删除 | NO | NO | NO |

> **谁是创始者？** 是程序首次启动时在"初始安装"屏幕中创建的管理员账户。
> 它会被自动指定，且该账户无法被删除或降级。
> 即使迁移到不同的计算机，受保护的账户始终保留为同一数据库中的第一个 admin。

### 示例场景

两位管理员（Ayşe = 创始者，Mehmet = 后期添加）在同一办公室工作：
- Mehmet 可以添加和删除 viewer 用户（具有完整权限）
- Mehmet 不能删除 Ayşe 或降低她的角色
- Mehmet 不能添加新管理员 — 此决定由 Ayşe 做出
- Ayşe 可以根据需要将 Mehmet 的角色降级为 viewer

## 2. 首次运行向导 (Setup Wizard) 与管理员账户

应用程序首次打开时会经历两阶段安装流程：

### 2a. 安装向导

登录界面之前会自动出现 **安装向导**。该向导通过 4 个步骤为用户准备系统：

1. **欢迎与系统检查**
   - 检查 WebAssembly (WASM) 支持
   - 显示操作系统版本和估算磁盘空间
   - 可进行**语言选择**（土耳其语 / 英语）— 因在登录界面之前提供，所以在此处出现

2. **硬件检测**
   - 显示 CPU 核心数、RAM 和基准测试结果
   - 系统自动推荐 低 / 中 / 高 性能模式
   - 如有需要可选择不同的模式

3. **AI 设置**
   - 自动检查 Ollama 服务器 (localhost:11434)
   - 如果 Ollama 在运行，则列出现有的视觉模型
   - 提供 3 种选择：
     - **本地 AI (Ollama)** — 数据保留在本地，推荐使用 GPU
     - **云端 AI (Gemini/Groq)** — 需要互联网和 API 密钥
     - **跳过 AI** — 之后可从设置中启用
   - 可使用"重新检查"按钮再次查询 Ollama 状态

4. **就绪！**
   - 总结所选的性能模式和 AI 模式
   - 通过"启动 Archivist Pro"按钮进入下一步

### 2b. 创建首个管理员账户

向导完成后，**如果之前从未创建过用户**，则不会显示登录界面，而是打开 **"初始安装"** 界面：

- 设置用户名和密码（用户名 3–32 个字符，密码最多 128 个字符）
- 点击"创建管理员账户"按钮
- 账户创建后自动登录

> **重要：** 系统中没有像 `admin / admin` 这样的预设账户。首个管理员账户由您自行创建。

### 旧用户

之前使用过应用程序的人（向导已完成，用户表中至少有 1 条记录）**不会再次看到**向导和初始安装界面。

### 重置向导

如果您出于测试目的想再次查看向导，请从浏览器开发者工具（F12 > Application > Local Storage）中删除 `archivist_setup_wizard_done` 和 `archivist_perf_setup_done` 键。

## 3. 密码恢复

如果您忘记管理员密码，可以使用应用程序启动时自动生成的 `recovery.key` 文件来重置密码。

### Recovery Key 文件

- 位置：`%APPDATA%\com.archivistpro.desktop\recovery.key`
- 48 个字符的十六进制字符串，应用程序首次打开时一次性生成
- 请将此文件备份到安全位置（U 盘、加密云盘等）

### 重置密码步骤

1. 在登录界面点击 **"忘记密码"** 链接
2. 将 `recovery.key` 文件的内容粘贴到输入框中
3. 选择要重置密码的管理员账户
4. 设置并确认新密码
5. 您将被重定向到登录界面

> **注意：** Recovery key 不是一次性的；每次重置都使用同一个 key。如果 key 文件丢失且所有密码都被忘记，则需要备份数据库文件并重置应用程序。

## 4a. V3 架构 (3.0.0+)

![V3 迁移面板](img/settings-v3-migration.png)

从 v3.0.0 开始，ArchivistPro 将档案数据存储在**两个独立的 SQLite 文件**中:

| 文件 | 内容 | 典型大小 |
|---|---|---|
| `archivist.db` | 资产元数据、标签、收藏集、设置、用户、关系类型 | 50-100 MB |
| `archivist_vec.db` | embedding 向量、文本片段、资产关联、FTS5 关键词索引 | 100-300 MB |

### 为什么分离？

- **性能:** 主 `archivist.db` 小 3-4 倍 → sql.js 加载和写入更快。实际测量: 181 MB → 52 MB。
- **内存:** sql.js 将数据库完全加载到 RAM。较小的主数据库 = 内存使用更少，阻塞性保存更少。
- **规模:** 数十万 embeddings 和数千文本片段存在于带有 HNSW ANN 索引的专用结构中；元数据数据库保持快速。
- **可维护性:** 向量数据可重新索引；元数据不可。分离使备份策略更加灵活。

### 自动迁移

来自早期版本的档案在首次启动时自动迁移:

1. 启动时读取 `PRAGMA user_version`。
2. 如果 `user_version < 3`，触发 `runV3EpochMigration`。
3. **预迁移备份:** `archivist.db` 复制到 `archivist_premigrate_v3.db.bak` (用于回滚)。
4. embeddings / text_chunks / relations 移至 `archivist_vec.db` (依次为 epoch 1 → 2 → 3)。
5. 验证: 源和目标的行数和内容是否完全匹配？
6. 成功时: 主数据库中的旧表通过 `DROP TABLE` + `VACUUM` + `PRAGMA user_version = 3` 删除。
7. 原子写入磁盘。

> 迁移在 **Rust 端**运行 (rusqlite)。180+ MB 的单体文件不会受到 Tauri IPC 负载限制的影响。

### 手动触发 (管理员面板)

如果自动迁移失败或首选管理员控制:

**设置 → 存储 → V3 架构迁移** 面板 (仅管理员):

- 显示当前 epoch (0/1/2/3) 和目标 (3)。
- "开始迁移"按钮 — 管理员触发的手动流程。
- 逐步进度: 预迁移 → 1/3 → 2/3 → 3/3 → 完成 → 重新加载。
- 错误时: 停止，显示消息，数据保持完整 (可通过 `.bak` 回滚)。

### 选择退出 (Opt-out)

可以通过标志禁用 V3 行为 (用于开发/测试):

```js
localStorage.setItem('ARCHIVIST_V3_EPOCH', 'off');
```

`off` 时不会触发自动迁移。**这仅在首次迁移前设置时有意义**；成功迁移后的回滚需要恢复 `.bak` 文件。

### 回滚

如果迁移后发现问题:

1. 关闭应用程序。
2. 移动或删除 `archivist_vec.db`。
3. 将 `archivist_premigrate_v3.db.bak` 重命名为 `archivist.db` (先备份当前 `archivist.db`)。
4. 重新打开应用程序 — 它返回到旧 (epoch=0) 状态。

### WAL Journal 和网络共享 (UNC)

v3.0.0 默认启用 WAL journal 模式 — 本地磁盘的写入性能提升约 30%。自动检测网络共享并回退到 DELETE 模式:

- **UNC 路径 (`\\server\share\archive\...`):** WAL 不安全 (NetBIOS 锁语义) → 强制 DELETE 模式。
- **映射的网络驱动器 (`Z:\archive\...` 但实际为 UNC):** 通过 `GetDriveTypeW` 检测，强制 DELETE 模式。
- **本地磁盘:** WAL 保持启用。

通过环境变量覆盖:

```cmd
set ARCHIVIST_DB_JOURNAL=delete
```

如果您积极在 UNC 上工作，**直接将共享打开为 UNC** (不使用映射驱动器号) — Windows NetBIOS 连接是一个边缘情况。对于较大的多用户设置，此模式是 HTTP LAN 共享的替代方案。

### 多档案 (阶段 1–3)
ArchivistPro 可以并行管理 N 个自定义档案。每个档案存储在独立的 SQLite 文件中，并拥有自己的源文件夹、标签和收藏夹。

**创建新档案：**
- 点击 设置 > 档案 > "新建档案" 按钮
- 输入名称、类型 (shared/personal) 和可选的磁盘路径
- 新档案会添加到左侧面板的选择器列表中

**合并 (Join/Merge)：**
- 将两个档案合并为一个档案
- 操作前会拍摄两个档案的快照
- 预览：将合并多少资源、冲突、标签、嵌入向量
- 冲突策略：skip / overwrite / rename
- 回滚：失败时完整还原

**提取 (Extract)：**
- 通过对档案应用筛选条件来创建子集
- 标准：类型、标签、类别、日期范围等
- 模式：copy（保留在源中）或 move（从源中删除）
- move 模式下源快照提供回滚保障

### 源文件夹管理（侧边栏面板）
左侧面板的"源文件夹"部分显示每个档案中已扫描的根目录：

- **添加：** 进行扫描时自动记录（每次扫描 = 单独一行，完整路径匹配）
- **重新扫描：** 通过三点菜单 — 仅以 scoped 方式重新扫描该文件夹，不影响其他文件夹
- **重命名：** 更改显示标签（路径不变）
- **移除：** 从列表中移除文件夹，资源仍保留在档案中
- **连同文件删除：** 文件夹 + 其下所有资源记录均被删除

> **注意：** 计数实时计算（不包括 BAK 和已删除的文件）。

### 扫描与索引
1. 点击左侧面板中的"扫描并索引文件夹"按钮
2. 选择模式：
   - **添加到列表**（默认）— 添加新文件
   - **从头扫描** — *仅*删除所选文件夹下的记录并重新扫描。其他源文件夹不受影响 (scoped replace)。
3. 扫描前会自动拍摄数据库快照（安全保障）
4. 保留最近 5 个快照，最旧的会自动删除
5. 扫描期间 **检查点 (checkpoint)** 系统启用 — 每 N 个文件（默认 50 个）将数据写入磁盘，即使发生崩溃/断电，已扫描的文件也不会丢失
6. 您可以从**设置 > 存储**选项卡更改检查点频率（每 1、5、10、25、50、75 或 100 个文件）

### 扫描报告
每次扫描后，被跳过或出错的文件列表会被保存。您可以通过**源文件夹 > 三点菜单 > "扫描报告"**访问这些报告。报告也以 TXT 格式保存到应用程序数据文件夹中。

### 文件夹变更检测 (Watch Folders)
如果您从设置中启用 **"监视文件夹变更"** 选项：
- 在已扫描的文件夹中添加/修改/删除文件时会收到通知
- 如果同时启用**自动重新扫描**选项，则在检测到变更时会自动重新扫描相应的文件夹
- 此功能仅适用于已开启的源文件夹

### DWG 元数据 — ODA File Converter 集成
要从 DWG 文件中提取真实的元数据（图层、块、文本内容、外部参照、绘图属性），需要使用 **ODA File Converter**。

**安装：**
- 设置 > AI > "ODA File Converter" 部分中自动检测已启用
- 如果已安装在系统上，会自动从 Registry + PATH 中找到
- 如果未安装，则提供"下载并安装"按钮（捆绑安装程序或 winget/web）

**行为：**
- 扫描期间，每个 DWG 文件都会在后台**不可见**地运行 ODA（通过 PowerShell `Start-Process -WindowStyle Hidden` 包装器，不打开窗口、不抢焦点）
- DWG → 转换为临时 DXF，使用 DXF 解析器提取元数据
- 结果写入 `dwgLayers`、`dwgBlockNames`、`dwgTextContents`、`dwgXrefNames` 字段
- **形状数据** — 提取文件中的几何形状（多段线、弧、矩形）并用于形状搜索
- **嵌入的 OLE 对象** — 检测嵌入到 DWG 内的 Excel/Word/PDF 文件
- 如果没有 ODA，会回退到原始二进制扫描（静默，元数据不完整）

**DWG 结构相似性搜索：**
当您右键点击一个 DWG 文件并选择"查找相似"时，使用 5 维**综合得分**而非 CLIP 视觉对比：图层结构、块结构、文本内容、形状数据和 pHash。该方法对 CAD 文件给出更可靠的结果。

### 文件重组 (Refile)
1. 点击顶部栏的"组织"按钮
2. 选择组织策略：
   - 按项目
   - 按类别
   - 按阶段
   - 按材料
3. 检查预览
4. 使用"应用"按钮移动文件

## 6. 备份与档案共享

### 数据库快照 (DB Snapshot)

快照是您数据库的完整备份。每次扫描/索引操作之前**自动**拍摄。此外，也可以手动拍摄。

> **v3.0.0 注意:** 快照现在**同时保存 `archivist.db` 和 `archivist_vec.db`**。恢复会原子地恢复两个文件 — V3 数据 (embeddings/chunks/relations) 不会从快照中丢失。

**自动行为：**
- 每次扫描开始前静默拍摄
- 保留最近 5 个快照，最旧的会自动删除

**手动快照（设置 > 存储）：**
1. 打开设置 → 切换到"存储"选项卡
2. 点击 **"创建备份"** 按钮 — 快照会以日期和大小添加到列表中
3. 每个快照行：
   - **恢复** — 用此快照替换当前数据库（需要确认）
   - **删除** — 仅永久删除此快照

> **注意：** Viewer 用户无法在主档案中拍摄快照。在自己的本地档案中具有完整权限。

### 档案导出/导入 (.archivistpro)

您可以将整个档案导出为单个 `.archivistpro` 文件，或导入来自其他计算机的档案。

**导出 (Export)：**
1. 转到设置 > 网络 选项卡
2. 点击"导出 / 报告"按钮
3. 选择保存位置和文件名
4. 档案以 `.archivistpro` 格式 (ZIP) 创建

**导入 (Import)：**
1. 转到设置 > 网络 选项卡
2. 点击"导入 (.archivistpro)"按钮
3. 选择要导入的文件
4. 显示清单 (manifest) 预览（版本、资源数量、数据库大小）
5. 确认 — 当前数据库会自动以 `.bak` 备份

### 局域网共享（小型 HTTP 服务器）

您可以通过办公室局域网与其他计算机共享档案。无需互联网连接。

**启动服务器：**
1. 点击 设置 > 网络 > "启动服务器" 按钮
2. 记下屏幕上显示的信息：
   - **IP 地址**（例如 `192.168.1.106`）
   - **端口**（`9471`）
   - **连接代码**（8 位数字，例如 `25930014`）
3. 通过口头/书面方式与 Viewer 用户分享这些信息

**安全：**
- 连接代码在第一次启动服务器时随机生成 (CSPRNG) 并**永久保存** — 即使重启应用程序，代码也不会改变
- 您可以通过点击"刷新代码"按钮手动刷新代码（新代码立即生效，无需重启服务器）
- 没有代码无法访问（403 错误）
- 5 次失败尝试后，源 IP 将被阻止 5 分钟
- 服务器仅在本地网络 (LAN) 中可访问
- 数据未加密 — 仅在受信任的办公室网络中使用

**停止服务器：**
- 点击"停止服务器"按钮
- 所有连接将被断开

> **注意：** 服务器运行时您无法切换到客户端模式（连接）。请先停止服务器。

## 7. 3ds Max 版本转换

### 快速模式（修改戳记）
- 修改文件的版本戳记
- 原始文件保留，旁边创建新文件
- 快速但在某些情况下可能出问题

### 真实模式 (MAXScript)
- 使用已安装的 3ds Max 重新保存文件
- 更可靠，但需要安装 Max
- Max 在后台 (headless) 运行

### FBX / OBJ 导出
您可以将 MAX 文件转换为 FBX 或 OBJ 格式。在详情面板中选择 MAX 文件时，会出现两个导出按钮：

| 模式 | 描述 | 要求 |
|------|------|------|
| **快速模式** | 基础几何转换 | 无 |
| **真实模式** | 通过 3ds Max 原生 FBXEXP/ObjExp 插件 | 已安装 3ds Max |

- 在真实模式下，您计算机上的 3ds Max 安装会被自动检测（Registry 扫描）
- 转换后的文件会保存到下载文件夹
- 操作期间 Max 在后台 (headless) 运行，应用 5 分钟超时

### MAX 元数据查看
在详情面板中为 MAX 文件显示额外信息：
- **图层** — 文件的图层结构（彩色标签）

## 9. 数据库安全

- 删除操作是原子的（受事务保护）
- 启用了外键级联 — 资源删除时，嵌入向量、标签、收藏夹会自动清理
- 数据库保存出错时会显示通知
- 更改数据库路径时应用路径遍历 (path traversal) 保护
- **v3.0.0:** 如果迁移或磁盘写入失败，流程停止并将内存状态与磁盘重新对齐 (防止假成功)
- **v3.0.0:** 每个档案的写入锁 — 同时使用多个档案时无写入冲突
- **v3.0.0:** 过时 WAL 清理 — blob-overwrite 后自动删除孤立的 `-wal`/`-shm` 文件

## 9f. AI 敏感性筛选器

防止档案中的敏感文件（合同、薪资表、个人数据）作为 AI 聊天结果出现的 3 层保护系统。

### 为何需要？

AI 聊天可以查询所有已扫描的文件。当 viewer 用户问"有客户合同吗？"或"找一下薪资表"时，如果档案中存在此类文件，AI 会找到并显示它。使用此筛选器，您可以将敏感数据对 AI 隐藏 — 文件继续在档案中显示，仅 AI 无法访问。

### 第 1 层 — 现成类别

从设置 > 安全 > **AI 敏感性筛选器** 卡片启用。可启用或禁用 4 个类别：

| 类别 | 检测的关键词 |
|------|------------|
| **财务 (Financial)** | 薪资、发票、报价、预算、付款、成本、进度款、估算、收入、支出、银行、IBAN |
| **个人信息** | 身份证号 (TC kimlik)、户籍、电话、地址、出生、驾照、护照 |
| **法律** | 合同、保密协议 (NDA)、保密、法庭、催告函、委托书、公证、诉讼 |
| **人力资源** | 员工档案、休假、考勤、绩效、纪律、招聘、面试 |

这些关键词会在文件名、项目名和文件内容（chunk text）中扫描。匹配的文件整体被排除在 AI 之外。

### 第 2 层 — 自定义关键字

您可以从同一设置卡片添加自己的关键字。例如：
- 敏感的客户名：`"Villa Kaya"`
- 机密的项目代码：`"internal"`、`"gizli"`

添加或删除关键字时，筛选器会立即更新。

### 第 3 层 — 手动文件/文件夹隐藏

- **单个文件：** 右键点击文件 → **"对 AI 隐藏"**（再次点击可撤销）
- **整个文件夹：** 在源文件夹面板的三点菜单中 → **"排除在 AI 之外"**

### 工作原理（技术）

```
用户向 AI 提问
    ↓
RAG pipeline 启动搜索 (FTS + 语义 + 元数据)
    ↓
敏感性筛选器介入：
  ✗ rag_excluded = 1 的文件 → 跳过
  ✗ 包含活动类别关键词的文件 → 跳过
  ✗ 包含自定义关键字的文件 → 跳过
    ↓
其余文件发送给 AI
```

> **注意：** 筛选器仅影响 AI 聊天。普通搜索、筛选、详情面板和重复查找器不受筛选器影响。

## 10. 重复与相似文件查找器

此工具检测、比较您档案中的重复或相似文件，并在必要时删除。

> **访问 — Admin：** 所有功能均启用（扫描、查看、**删除**）。
> **访问 — Viewer：** 可以扫描和查看，但**删除操作仅对 admin 开放**。

### 10.1 访问面板

点击顶部栏中的 **⎇ (fork/branch)** 图标 — 它位于回收站图标和大脑图标之间。悬停时会显示"查找重复"。

### 10.2 检测模式

四种独立模式可以一起或分别工作：

| 模式 | 检测内容？ | 工作原理？ |
|------|---------|-----------|
| **完全重复** | 内容相同的文件（可能名称/位置不同） | 文件哈希 (SHA 比较) — 即时 |
| **同名** | 不同文件夹中的同名文件 | 文件名匹配 — 即时 |
| **视觉相似** | 视觉上相似的图像 | pHash Hamming distance (64-bit) — 约 100ms/1000 张图像 |
| **结构相似** | 相似的图层/材料/内容结构 | Jaccard similarity — 仅 CAD、3D、文档 |

#### 支持的文件类型（视觉相似）
`JPG · PNG · BMP · WEBP · TIFF · TGA · EXR · HDR · PSD`

#### 支持的文件类型（结构相似）
`DWG · DXF · IFC · MAX · SKP · PDF · DOC/DOCX · XLS/XLSX · PPT/PPTX · RVT`

#### 结构相似详情（按文件类型，v2.1.2）

| 类型 | 比较的字段 |
|------|-----------|
| DWG / DXF | 图层名 (1.0) + 块名 (1.0) + 文本内容 (0.8) + 外部参照 (0.6) — 加权 Jaccard |
| IFC | 楼层数 + 实体数（接近度匹配）+（如有）图层名 |
| 3DS MAX | 材质名 (Jaccard) + 渲染引擎匹配 (+50) + Max 版本匹配 (+35) |
| SketchUp | 组件名 + 图层名 (Jaccard) + SketchUp 版本匹配 (+35) |
| Revit | 楼层名 (Jaccard) + 项目名 (+50) + 面积数接近度 |
| PDF / DOCX / XLSX... | 标题 (+40) + 作者 (+30) + 页数 (+30) |

#### 通用条件（跨格式预筛选）
对结构相似性和同名模式可选的额外条件：

- **相同文件大小** — 可选容差：完全匹配、±1 KB、±1%
- **修改日期接近** — N 天窗口
- **相同文件夹名** — parent 文件夹 basename 匹配（不区分大小写）

这些条件通过 `passesGeneralCriteria()` 在文件对级别应用，不匹配的对甚至不会进入相似度计算（减少误报 + 加速）。

#### 性能筛选器
- **最小文件大小 (KB)** — 低于此大小的文件会通过预筛选从扫描池中排除

## 13. 回收站与恢复

删除的文件不会被永久删除，而是先移到回收站（软删除）。

### 移到回收站
- 文件被删除时被标记为 `is_deleted = 1`
- 所有元数据、标签、关系和项目状态信息都会保留

### 恢复
- 点击顶部栏中的回收站图标
- 从列表中选择文件并点击"恢复"按钮
- 文件连同所有信息一起返回到档案中

### 永久删除
- 通过回收站中的"永久删除"将文件从数据库中完全移除
- 通过"清空回收站"将回收站中的所有文件永久删除
- 此操作无法撤销

> **注意：** 在重新扫描期间，如果回收站中的某个文件的同一份在磁盘上被找到，则该文件会自动从回收站取出并返回到档案中（用户定义的字段会保留）。

此保护可防止长时间扫描或下载操作被意外中断。
