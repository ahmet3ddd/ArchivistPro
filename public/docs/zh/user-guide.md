# ArchivistPro 用户指南

> 版本 3.0.0 | 2026-05-23

## 欢迎使用

ArchivistPro 是一款桌面应用程序，用于智能归档、搜索和管理您的建筑设计文件。**完全离线运行** — 您的文件不会离开您的计算机。

![主窗口](img/main-window.png)

> **升级到 v3.0.0:** 如果您来自较早版本，首次启动时您的档案将**自动迁移到 V3 架构**。此过程需要几秒钟，无需您操作。为回滚目的，会在档案旁边保存一个名为 `archivist_premigrate_v3.db.bak` 的备份文件。详情请参阅下方"数据安全"部分。

---

## 1. 首次启动 — 安装向导

首次打开应用程序时，**安装向导 (Setup Wizard)** 会引导您完成 4 个步骤的系统准备工作：

1. **系统检查** — 测试您计算机的兼容性并选择显示语言（土耳其语和英语完全支持；中文/日语/阿拉伯语界面部分翻译）
2. **硬件检测** — 分析您的 CPU 和内存，并推荐合适的性能模式（低 / 中 / 高）
3. **AI 设置** — 选择 AI 功能的工作方式：
   - **本地 AI (Ollama)** — 所有数据保留在您的计算机上
   - **云端 AI** — 通过互联网工作（需要 API 密钥）
   - **跳过** — 不启用 AI 直接开始使用，稍后可在设置中启用
4. **总结** — 显示您的选择并完成设置

> **注意：** 向导仅显示一次。在首次运行的向导之后，系统会提示您创建管理员账户（没有预设密码 — 由您自行设定）。后续启动将直接进入登录界面。

## 2. 主界面

主界面由三个区域组成：

- **左侧面板（侧边栏）：** 顶部为档案选择器和收藏夹，中部为搜索框和视觉搜索灵敏度，底部为**源文件夹 (Source Folders)** 和分面筛选器（类别、项目阶段、材质等）
- **中央区域：** 文件列表（资源管理器、仪表板或技术视图）
- **右侧面板（详情）：** 所选文件的详细信息

### 视图模式

| 模式 | 描述 |
|------|------|
| 资源管理器 (Explorer) | 以卡片网格视图显示文件 |
| 仪表板 (Dashboard) | 统计与分析图表 |
| 技术视图 (Technical) | 表格形式的详细列表 |

### 源文件夹面板

![源文件夹](img/sidebar-source-folders.png)

左侧面板的语义搜索区域下方就是**源文件夹**面板。在此处：

- **您扫描的每个根文件夹都会自动列出**（按文件夹名称及该文件夹中的文件数量显示）
- 点击文件夹**将其作为筛选条件**应用 — 仅显示该文件夹中的文件
- 可选择多个文件夹（OR 逻辑：A 或 B 或 C）
- 使用"清除筛选"移除所有筛选
- 点击文件夹名称左侧的**▶ 箭头**可展开子文件夹树 — 显示嵌套的文件夹结构；点击任何子文件夹即可按其筛选
- 每个文件夹旁边的**三点菜单**提供以下操作：
  - **重命名** — 更改文件夹的显示标签
  - **重新扫描** — 仅重新扫描该文件夹（不影响其他源文件夹）
  - **扫描报告** — 在先前扫描中被跳过或出错的文件列表
  - **移除** — 从列表中移除文件夹（文件保留在档案中）
  - **连同文件删除** — 移除文件夹 + 其所有资源记录

> **注意：** 当文件被删除或运行新扫描时，计数实时更新。BAK（备份）文件不计入数量，也不会显示在主网格中。

## 3. 文件扫描与索引

1. 点击左侧面板中的 **"扫描并索引文件夹 (Scan & Index Folder)"** 按钮
2. 选择要扫描的文件夹
3. 设置扫描模式：
   - **添加到列表（默认）：** 新文件添加到现有扫描中
   - **从头重新扫描（替换）：** *仅*删除所选文件夹下的旧记录并重新扫描。**其他源文件夹不受影响。**
   - **颜色提取：** 分析图像的主色调（可选）
4. 点击 **"开始扫描"**

扫描期间，模态框标题显示 **"扫描并索引文件 (Scan & Index Files)"**。

- 您可以跟踪进度条和速度
- **暂停/继续**临时停止扫描
- **取消**完全终止扫描

> **关于 DWG 文件：** 如果安装了 ODA File Converter，则会自动提取图层、块、文本内容和外部参照，并显示在详情面板中。ODA 在后台不可见运行。

> **要刷新单个源文件夹，** 在"源文件夹"面板中打开该文件夹的三点菜单并选择 **"重新扫描"** — 模态框自动打开，仅对该文件夹运行范围扫描。

### 数据安全（检查点 + V3 备份）

**V3 档案文件结构 (v3.0.0 及更高版本):**

您的档案现在由两个独立的文件组成：

- **`archivist.db`** — 主数据库。资产元数据、标签、收藏集、设置。
- **`archivist_vec.db`** — 向量数据库。embeddings、文本片段 (chunks)、文件关联、FTS5 关键词索引。

这种拆分使主数据库尺寸减少 3-4 倍，并显著提高大型档案的性能。**两个文件保存在同一文件夹中**，备份时必须**一起复制**。内置的备份/恢复流程（快照、档案导出/导入）会自动处理两个文件。

> **迁移备份:** 首次迁移到 V3 时，旧的单文件档案的副本将保存为 `archivist_premigrate_v3.db.bak`。如果出现问题，您可以将此文件复制回 `archivist.db` 以恢复之前的版本（需先删除 `archivist_vec.db`）。确认一切正常后，可以安全删除 `.bak` 文件。

### 数据安全（检查点 Checkpoint）

扫描期间，您的数据会定期保存到磁盘。即使发生断电或崩溃，已扫描的文件也不会丢失。您可以从**设置 > 存储**更改检查点频率（默认：每 50 个文件）。

### 文件夹变更检测 (Watch)

如果您在设置中启用 **"监视文件夹变更"**，当扫描的文件夹中添加或修改文件时，应用程序会通知您。您也可以启用自动重新扫描。

### 多核扫描与硬件使用

ArchivistPro 在多个 CPU 核心上并行运行扫描操作。在初始设置期间，应用程序会自动检测您的存储类型并选择合适的默认工作线程数：

| 存储类型 | 推荐工作线程数 |
|---|---|
| HDD（机械硬盘） | 1 – 2 |
| SSD (SATA) | 3 – 4 |
| NVMe（≤ 8 个逻辑核心） | 6 – 8 |
| NVMe（≥ 16 个逻辑核心） | 10 – 16 |

您可以从**设置 > 存储**更改工作线程数。在 HDD 驱动器上，过高的工作线程数不会提高性能；相反，由于磁盘磁头争用，可能会导致速度变慢。

> **关于 AI 功能（语义搜索、RAG）：** 使用 GPU 而非 CPU。这些操作通过单独的 Ollama 服务运行，不会减慢文件扫描速度。

## 4. 搜索

### 文本搜索
在搜索框中开始输入。系统会执行三层搜索：
- **关键字匹配：** 文件名、项目名、元数据、客户名、审批状态
- **语义搜索：** 通过 AI 进行基于含义的匹配（至少需要 3 个字符）
- **建筑专业术语：** 术语会自动扩展（例如 "kitchen" → 相关概念）
- **模糊搜索：** 容忍拼写错误 — 拼错的单词仍能返回正确结果（4 个或更多字符的单词，最多 30% 的错误容差）

> **提示：** 您可以搜索包含连字符的文件代码 — 像 "A1-c3" 或 "A1-" 这样的短代码可以直接匹配。

### 布尔搜索（高级）

您可以在搜索框中使用逻辑运算符：

| 运算符 | 示例 | 含义 |
|--------|------|------|
| **AND** | `plan AND section` | 同时包含两者的文件 |
| **OR** | `kitchen OR bathroom` | 至少包含一个的文件 |
| **NOT** | `project NOT old` | 包含 "project" 但不包含 "old" |
| **"引号"** | `"floor plan"` | 完全包含此短语（单词相邻并按顺序） |

> 运算符必须以**大写**书写（AND 而非 and）。可以组合使用：`"floor plan" AND facade NOT draft`

### 高级搜索菜单

点击顶部栏中的 **"高级搜索 (Advanced Search)"** 按钮可访问其他搜索类型：
- **视觉搜索 (Visual Search)** — 上传图像以查找相似的视觉内容
- **形状搜索 (Shape Search)** — 在 DWG/DXF 文件中按几何形状搜索
- **查找相似 (Find Similar)** — 列出与所选文件相似的文件
- **重复查找器 (Duplicate Finder)** — 检测重复文件

### 视觉搜索
通过左侧面板的图像上传按钮或高级搜索菜单上传图像。系统会自动查找相似的视觉内容。

### 排序

您可以按不同标准对结果进行排序：
- **匹配度**（默认）— 搜索结果按相关性从高到低排列
- **修改日期** — 最新或最旧优先
- **文件名** — 按字母顺序
- **文件大小** — 从大到小或反之

### 筛选器
使用左侧面板中的分面来缩小结果范围：
- **类别：** 2D 图纸、3D 模型、文档、渲染图、照片、贴图、视频
- **项目阶段：** 概念、方案、报建、施工
- **审批状态：** 草稿、审核中、已批准、已拒绝（在项目状态部分分配）
- **材料组：** 混凝土、玻璃、金属、木材、石材、陶瓷、复合材料
- **色彩主题：** 暖色调、冷色调、单色、大地色调、柔和色调
- **建筑风格：** 现代、极简、工业、粗野主义、新古典、有机

### 日期范围筛选

左侧面板的分面下方有一个**日期范围筛选器**。选择开始和结束日期以按修改日期缩小文件范围。例如，可仅查看最近 3 个月内更改的文件。

### 筛选预设 (Filter Presets)

您可以将常用的筛选组合保存为**预设**：
1. 按需设置筛选器（分面 + 标签 + 日期 + 搜索词）
2. 点击筛选栏中的 **"保存"** 按钮
3. 为预设命名
4. 之后可一键重新加载相同的筛选条件

> 可勾选多个选项（OR 逻辑）。筛选器会即时应用 — 无需重新扫描。

## 5. 文件详情

点击文件即可在右侧面板中查看其详细信息：
- 预览（缩略图或原生视图）
- 调色板（Hex、RGB、HSL、RAL 代码）
- 元数据（大小、日期、图层、块）
- AI 标签
- 用户标签
- 项目状态字段
- 关联文件

### 5.1 项目状态

详情面板包含一个可输入文件项目信息的区域：

| 字段 | 描述 | 限制 |
|------|------|------|
| **客户 (Client)** | 与文件关联的客户或公司名称 | 最多 150 个字符 |
| **审批状态** | 选择 4 种状态按钮之一 | 草稿 / 审核中 / 已批准 / 已拒绝 |
| **版本 (Version)** | 版本标签（例如 v1.0、Rev-A） | 最多 20 个字符 |
| **交付日期** | 从日历中选择或手动输入 | ISO 日期格式 |

- 点击任何字段进入编辑模式；按 Enter 键或点击外部以保存
- 接近字符限制时，计数器变为橙色（客户：140+、版本：18+）
- 即使重新扫描文件，**这些数据也会保留** — 扫描器不会触碰这些字段
- 客户名和审批状态**会影响搜索结果**，并可在侧边栏中筛选

#### 拒绝原因
当您将文件的审批状态更改为 **"已拒绝"** 时，底部会出现一个文本框。您可以输入拒绝原因（例如"尺寸有误，需要修订"）。当文件后来被批准时，拒绝原因会自动清除。

#### 审批历史
每次审批状态变更都会被记录。管理员可在仪表板视图的 **"审批历史"** 面板中按时间顺序查看最近的操作（谁、何时、从哪个状态变更到哪个状态以及原因，如有）。

### 5.2 关联文件（文件关系）

详情面板中的"关联文件"部分显示文件与其他文件的关系：

| 类型 | 含义 | 示例 |
|------|------|------|
| **PDF 输出** | 此文件的 PDF 版本 | plan.dwg ↔ plan.pdf |
| **渲染** | 此设计的可视化 | salon.max ↔ salon_render.jpg |
| **版本** | 同一文件的不同版本 | plan_v1.dwg ↔ plan_v2.dwg |
| **项目组** | 属于同一项目的文件 | Tower_A/*.* |

- **自动检测：** 同一文件夹中名称相同但扩展名不同的文件（例如 plan.dwg + plan.pdf）会在扫描后自动关联
- **自动版本聚类：** 在扫描期间，名称相似的文件（例如 `plan_v1.dwg`、`plan_v2.dwg`、`plan_Rev-A.dwg`、`plan_FINAL.dwg`）会自动以"版本"关系链接。识别的模式：`_v1`、`_Rev-A`、`_R01`、`_FINAL`、`_DRAFT`、`_old`、`_new`、`(Copy)`、末尾数字
- **手动关联：** 使用"添加关联"按钮在任意两个文件之间创建关系
- **导航：** 点击关联文件的名称可导航到其详细信息
- 自动检测的关联会标记 `[Auto]` 标签

### 5.3 格式特定信息

#### 3ds MAX (.max) 文件
- **图层** — 文件的图层结构以彩色标签显示
- **对象** — 列出文件中的对象名称（超过 30 个时附加 "+N more" 提示）
- **FBX / OBJ 导出** — 详情面板中显示两个转换按钮：
  - **快速模式：** 无需 3ds Max，进行基础几何转换
  - **原生模式：** 如果您的计算机上安装了 3ds Max，则进行全质量转换（自动检测）

#### Revit (.rvt) 文件
- 无需 3ds Max 即可自动提取缩略图预览（来自文件中嵌入的 OLE 流）
- 显示项目名、Revit 版本和楼层名称

#### DWG 文件（搭配 ODA）
- 图层、块、文本内容、外部参照
- 图纸类型（平面图、剖面图、立面图等 — 通过 AI）
- 比例与单位估算
- **形状数据** — 提取文件中的几何形状（矩形、圆、多段线等）并用于形状搜索
- **结构相似性** — 右键点击 DWG 并选择"查找相似"会通过比较图层/块/文本/形状结构对最相似的 DWG 文件进行排名（使用结构综合得分而非 CLIP 视觉对比 — 对 CAD 文件更可靠）
- **嵌入的 OLE 对象** — 检测嵌入在 DWG 内的 Excel、Word 或 PDF 文件并显示在详情面板中

## 9. AI 功能

### AI 状态指示器

顶部栏右侧有一个**大脑图标**，旁边带有一个彩色圆点：

| 圆点颜色 | 含义 |
|----------|------|
| 绿色 | AI 就绪 — 模型已加载并运行 |
| 黄色 | 缺少模型 — AI 设置未完成 |
| 红色 | Ollama 已关闭或无连接 |

将鼠标悬停在大脑图标上会显示 **"打开 AI 安装向导"** 链接。点击该图标打开 AI 设置模态框。

---

### AI 安装向导

首次设置 AI 功能或进行故障排除：

1. 点击顶部栏中的**大脑图标** → AI 设置 → **"AI 安装向导"** 按钮
   *或者* 将鼠标悬停在大脑图标上 → 点击 **"打开 AI 安装向导"** 链接

2. 向导通过 **3 个步骤**完成设置：
   - **1. Ollama 检查** — 检查 Ollama 是否已安装并运行。如果未安装，可使用"启动"按钮直接从应用程序启动它，或从 [ollama.com](https://ollama.com) 下载。
   - **2. 下载模型** — 下载所需的 AI 模型（文本和/或视觉）。可以使用进度条跟踪下载进度。
   - **3. 完成** — 系统已就绪。AI 聊天和自动分类功能将激活。

---

### 启动 / 停止 Ollama

您可以从 AI 设置面板手动管理 Ollama 服务：

- **如果 Ollama 已关闭** → 出现绿色 **"启动"** 按钮（电源图标）。点击它会在后台启动 `ollama serve`。
- **如果 Ollama 已开启** → 出现红色 **"停止"** 按钮。点击它会停止该服务。
- 状态变更期间，按钮显示"正在启动..." / "正在停止..."。

> **注意：** Ollama 关闭时会隐藏高级模型设置部分 — 这是正常现象。

### AI 聊天 (RAG)

![AI 聊天窗口](img/ai-chat-empty.png)

点击顶部栏中的 **"💬 AI"** 按钮打开 AI 聊天面板。您可以使用自然语言询问有关档案的问题。

**v3.0.0 注意:** 是/否问题（例如"有 X 吗？""包含 Y 吗？"）将直接以文件列表的形式立即回答 — 无需等待 LLM。

**聊天设置 (⚙ 齿轮图标):**

![AI 聊天设置](img/ai-chat-settings.png)

聊天窗口右上角的齿轮图标显示以下设置:
- **模型 (Model)** — 使用哪个 Ollama 模型（例如 `qwen2.5:8b`）
- **Ollama 状态** — 服务是否运行
- **元数据索引 (Meta indeks)** — 档案中有元数据索引的文件数
- **内容 (İçerik)** — 有文本索引的文件数
- **重新排序 (Rerank)** — 由 AI 重新排序结果（提高质量，约慢 10%）
- **扩展 (Zenginleştir)** — 用同义词扩展查询（提高召回率）
- **思考 (Düşünme)** — 显示模型的思考步骤（实验性）

**主要功能：**

- **带引用的回答 (Citation)：** 每个 AI 响应都会显示使用了哪些文件。点击源文件名可直接跳转到该文件的详细信息。
- **范围筛选器：** 从聊天窗口顶部缩小搜索范围 — 可在特定项目、标签或整个档案之间选择。
- **流式响应：** AI 答案在生成时实时流式传输到屏幕；无需等待较长的响应。

**示例问题：**
- "有带木质立面的渲染图吗？"
- "列出 2024 年所有已批准的平面图"
- "查找客户名为 'Smith' 的项目中的 PDF 文件"

### AI 敏感性筛选器（管理员）

档案中的某些文件可能包含敏感信息（薪资表、合同、个人数据）。如果您不希望这些文件出现在 AI 聊天的结果中，请使用 **AI 敏感性筛选器**。

**为何需要？** AI 聊天会搜索所有已扫描的文件。如果用户问"有薪资表吗？"，而档案中存在此类文件，AI 将找到并显示它。该筛选器会向 AI 隐藏此类文件。

**3 种保护方式：**

1. **现成类别** — 从设置 > 安全 > AI 敏感性筛选器启用：
   - **财务**：薪资、发票、报价、预算、进度款...
   - **个人信息**：身份证号、电话、地址...
   - **法律**：合同、保密协议 (NDA)、委托书...
   - **人力资源**：员工档案、休假、考勤、绩效...

2. **自定义关键字** — 添加您自己的关键字（例如敏感的客户名、项目代码）

3. **按文件/文件夹** — 右键点击文件 → "对 AI 隐藏"，或在源文件夹菜单中 → "排除在 AI 之外"

> **注意：** 隐藏的文件仍在档案中显示 — 搜索、筛选和详情面板正常工作。仅 AI 聊天无法访问这些文件。

## 12. 重复与相似文件查找器

此工具显示档案中重复或相似的文件。

> **作为 viewer：** 您可以运行扫描并查看结果。**文件删除仅限管理员** — 不会显示删除按钮。

### 访问面板

![重复查找器](img/duplicate-finder.png)

点击顶部栏中的 **⎇** 图标（位于回收站和大脑图标之间）。悬停时显示"查找重复 (Find Duplicates)"。

### 范围选择

面板打开时，从选项卡中选择要扫描的档案（列出主档案、本地档案及您添加的所有自定义档案）。

> 如果尚未加载本地档案，则该选项卡显示为非活动状态。从侧边栏切换到本地档案后再回来。

> **关于大型档案**（2000+ 个文件），开始扫描前会显示警告 — 扫描时间可能因所选模式而异。扫描期间您可以使用**取消**按钮立即停止扫描。

### 检测模式

| 模式 | 检测内容 |
|------|---------|
| **完全重复** | 内容相同的文件（SHA-256 哈希匹配 — 可能名称/文件夹不同） |
| **同名** | 不同文件夹中的同名文件（可使用通用条件缩小范围 — 例如同名 + 同大小） |
| **视觉相似** | 视觉上相似的照片/渲染/图像文件 (pHash) |
| **结构相似** | 具有相似图层、材料或内容结构的 CAD/3D/文档文件（Jaccard + 综合得分） |

### 相似度阈值

使用滑块为**视觉**和**结构**相似度设置灵敏度：

- **低 %** → 更多、宽松匹配的结果（可能存在误报）
- **高 %** → 更少、仅非常相似的结果

### 高级条件面板

点击"高级条件"会打开一个具有 **4 个部分**的面板：

#### 1. 通用条件（跨格式预筛选）
为同名和结构相似模式添加额外条件：

- **相同文件大小** — 容差：完全匹配、±1 KB 或 ±1%
- **修改日期接近** — N 天窗口（1–365）
- **相同文件夹名** — 父文件夹名匹配（不区分大小写）

> 示例："同名 + 同大小"是非常强烈的重复信号 — 显著减少误报。

#### 2. 格式特定条件
选择在结构相似性中比较哪些元数据字段。按格式分组：

- **DWG / DXF**：图层 · 块 · 文本内容 · 外部参照 (Xrefs)
- **IFC**：楼层数 · 实体数
- **3DS MAX**：材质列表 · 渲染引擎 · Max 版本
- **SketchUp**：组件 · 图层 · SketchUp 版本
- **Revit**：楼层名 · 项目名
- **PDF / Office**：标题 · 作者 · 页数

#### 3. 性能筛选器
缩小扫描池以提高速度：

- **最小文件大小 (KB)** — 低于此大小的文件完全排除在扫描之外

#### 4. 格式可见性筛选器
实时调整哪些文件类别出现在扫描结果中（无需重新扫描）：

CAD · BIM/3D · 文档 · 视觉 · 视频 · 备份

> 当通用条件或性能设置发生变化时，面板底部会出现"需要重新扫描"警告。

### 查看结果

**列表视图：**

```
▼ VISUAL SIMILAR — 1 group
  ┌─────────────────────────────────────────────────┐
  │ render_v1.jpg   pHash diff: 5/64 bits → 92%    │
  │  /Render/render_v1.jpg   2.4 MB   2024-03-01   │
  │  /Render/render_v2.jpg   2.4 MB   2024-03-15   │
  └─────────────────────────────────────────────────┘
```

点击组以展开/折叠。显示每个文件的路径、大小和日期。

### 比较视图

要并排检查两个文件，点击 `[⎇]` 图标，然后切换到顶部的 **[Compare]** 按钮：

```
┌───────────────────┬───────────────────┐
│   render_v1.jpg   │   render_v2.jpg   │
│  [ preview ]      │  [ preview ]      │

│  2024-03-01       │  2024-03-15       │
├───────────────────┴───────────────────┤
```
